# Antigravity Manager 使用教程

Antigravity Manager 是一个 AI 账号管理与协议代理网关。它能管理多个 Google 账号，将请求统一转换为 OpenAI / Claude / Gemini 兼容格式，并在多账号间智能轮换和负载均衡。

---

## 1. 启动服务

本项目通过 Docker 无头模式运行（Rust 后端 + React 前端打包在同一镜像中）。

### 首次启动

```bash
# 拉取镜像
docker pull lbjlaq/antigravity-manager:latest

# 创建数据目录
mkdir -p ~/.antigravity_tools

# 启动容器
docker run -d \
  --name antigravity-manager \
  -p 8045:8045 \
  -e API_KEY=test \
  -e WEB_PASSWORD=pwd \
  -e LOG_LEVEL=info \
  -v ~/.antigravity_tools:/root/.antigravity_tools \
  --restart unless-stopped \
  lbjlaq/antigravity-manager:latest
```

### 日常操作

```bash
# 查看日志
docker logs -f antigravity-manager

# 停止
docker stop antigravity-manager

# 重启
docker restart antigravity-manager

# 删除并重建（保留数据，因为挂载了 ~/.antigravity_tools）
docker rm -f antigravity-manager
# 然后重新执行上面的 docker run 命令
```

### 启动后确认

```bash
curl http://127.0.0.1:8045/api/health
# 返回：{"status":"ok","version":"4.1.26"}
```

浏览器访问 **http://127.0.0.1:8045**，用密码 `pwd` 登录 Web 管理面板。

## 2. 添加 Google 账号（首次必做）

没有账号，API 无法调用。启动后日志会提示"没有可用账号"。

### 方式一：通过 Web 面板 OAuth 授权

1. 打开 **http://127.0.0.1:8045**
2. 进入"账号管理"页面
3. 点击"添加账号"，选择 OAuth 授权
4. 跳转到 Google 登录页，完成授权
5. 授权成功后账号自动添加到列表

### 方式二：通过 API 导入 Refresh Token

如果你已经有 Google OAuth Refresh Token：

```bash
curl -X POST http://127.0.0.1:8045/api/accounts \
  -H "x-admin-token: test" \
  -H "Content-Type: application/json" \
  -d '{"refreshToken": "你的refresh_token"}'
```

### 刷新账号配额

添加账号后，刷新配额信息：

```bash
curl -X POST http://127.0.0.1:8045/api/accounts/refresh \
  -H "x-admin-token: test"
```

## 3. 鉴权体系

本服务有两套鉴权：

| 类型 | 路径 | 鉴权方式 | 用途 |
|------|------|----------|------|
| AI 协议接口 | `/v1/*` | `Authorization: Bearer <API_KEY>` | 给 AI 客户端调用 |
| 管理接口 | `/api/*` | `x-admin-token: <API_KEY>` | 管理账号、查看日志等 |

默认 API_KEY 为启动时设置的 `test`，Web 登录密码为 `pwd`。

## 4. 配置 AI 客户端

### 4.1 作为 OpenAI API 使用（最常用）

适用于 ChatGPT-Next-Web、LobeChat、Open WebUI、Cursor 等客户端。

| 配置项 | 值 |
|--------|------|
| API Base URL | `http://127.0.0.1:8045/v1` |
| API Key | `test` |
| 模型 | `gemini-2.5-pro` 或其他映射模型 |

**curl 测试**：

```bash
curl http://127.0.0.1:8045/v1/chat/completions \
  -H "Authorization: Bearer test" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini-2.5-pro",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": false
  }'
```

**Python (openai 库)**：

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:8045/v1",
    api_key="test",
)

response = client.chat.completions.create(
    model="gemini-2.5-pro",
    messages=[{"role": "user", "content": "你好"}],
)
print(response.choices[0].message.content)
```

### 4.2 作为 Claude API 使用

适用于 Claude Code CLI、Cherry Studio 等支持 Anthropic 协议的客户端。

```bash
# 设置环境变量后直接使用 Claude CLI
export ANTHROPIC_BASE_URL="http://127.0.0.1:8045/v1"
export ANTHROPIC_API_KEY="test"

claude "你好"
```

或用 curl：

```bash
curl http://127.0.0.1:8045/v1/messages \
  -H "Authorization: Bearer test" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini-2.5-pro",
    "max_tokens": 1024,
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

### 4.3 作为 Gemini 原生 API 使用

适用于 Google AI SDK。

```bash
curl -X POST "http://127.0.0.1:8045/v1beta/models/gemini-2.5-pro:generateContent" \
  -H "Authorization: Bearer test" \
  -H "Content-Type: application/json" \
  -d '{"contents": [{"role": "user", "parts": [{"text": "你好"}]}]}'
```

## 5. 图片生成

使用 OpenAI Images API 格式调用 Gemini 3 Pro Image：

### 文生图

```bash
curl http://127.0.0.1:8045/v1/images/generations \
  -H "Authorization: Bearer test" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini-3-pro-image",
    "prompt": "赛博朋克城市夜景，霓虹灯光",
    "size": "16:9",
    "quality": "hd",
    "n": 1
  }'
```

### 画质参数

| quality 值 | 说明 |
|------------|------|
| `standard` / `1k` | 快速出图，适合预览（默认） |
| `medium` / `2k` | 平衡画质与速度 |
| `hd` / `4k` | 最高画质 |

### 比例参数

`size` 支持直接填比例（如 `"16:9"`）或分辨率（如 `"1920x1080"`，会自动映射到最近的标准比例）。

支持的比例：`1:1`、`16:9`、`9:16`、`4:3`、`3:4`、`21:9`、`3:2`、`2:3`、`5:4`、`4:5`

### 图生图

```bash
curl http://127.0.0.1:8045/v1/images/edits \
  -H "Authorization: Bearer test" \
  -F "prompt=基于这张图生成水彩风格" \
  -F "image1=@/path/to/reference.jpg" \
  -F "aspect_ratio=16:9" \
  -F "image_size=4K"
```

## 7. 管理接口速查

所有管理接口需要 `x-admin-token` 头。

### 账号管理

```bash
# 查看所有账号
curl http://127.0.0.1:8045/api/accounts \
  -H "x-admin-token: test"

# 查看当前活跃账号
curl http://127.0.0.1:8045/api/accounts/current \
  -H "x-admin-token: test"

# 刷新所有账号配额
curl -X POST http://127.0.0.1:8045/api/accounts/refresh \
  -H "x-admin-token: test"

# 删除账号
curl -X DELETE http://127.0.0.1:8045/api/accounts/<account_id> \
  -H "x-admin-token: test"
```

### 代理控制

```bash
# 查看代理状态
curl http://127.0.0.1:8045/api/proxy/status \
  -H "x-admin-token: test"

# 启动代理
curl -X POST http://127.0.0.1:8045/api/proxy/start \
  -H "x-admin-token: test"

# 停止代理
curl -X POST http://127.0.0.1:8045/api/proxy/stop \
  -H "x-admin-token: test"
```

### 统计与日志

```bash
# Token 消耗摘要
curl http://127.0.0.1:8045/api/stats/token/summary \
  -H "x-admin-token: test"

# 按模型统计
curl http://127.0.0.1:8045/api/stats/token/by-model \
  -H "x-admin-token: test"

# 查看日志（最近 50 条）
curl "http://127.0.0.1:8045/api/logs?limit=50" \
  -H "x-admin-token: test"
```

## 8. 环境变量配置

启动容器时通过 `-e` 设置：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `API_KEY` | 随机生成 | AI 接口和管理接口的鉴权密钥 |
| `WEB_PASSWORD` | 同 API_KEY | Web 面板登录密码 |
| `PORT` | `8045` | 服务端口 |
| `LOG_LEVEL` | `info` | 日志级别：debug/info/warn/error |
| `ABV_MAX_BODY_SIZE` | `104857600` | 最大请求体（100MB） |
| `ABV_BIND_LOCAL_ONLY` | 空 | 设置后仅绑定 127.0.0.1 |
| `ABV_PUBLIC_URL` | 空 | 外部访问 URL（用于 OAuth 回调） |

### 修改配置示例

```bash
docker rm -f antigravity-manager

docker run -d \
  --name antigravity-manager \
  -p 8045:8045 \
  -e API_KEY=my-secret-key \
  -e WEB_PASSWORD=my-web-password \
  -e LOG_LEVEL=debug \
  -v ~/.antigravity_tools:/root/.antigravity_tools \
  --restart unless-stopped \
  lbjlaq/antigravity-manager:latest
```

## 9. 数据持久化

所有数据存储在 `~/.antigravity_tools/` 目录，通过 Docker Volume 挂载：

- `gui_config.json` — 账号信息、API Key、系统配置
- SQLite 数据库 — 日志、统计数据

只要保留这个目录，删除容器重建不会丢失数据。

## 11. 常见问题

**Q: 启动后报"没有可用账号"？**
正常现象，需要先添加 Google 账号（见第 2 节）。

**Q: macOS Docker 访问不了 8045 端口？**
确认使用了 `-p 8045:8045` 端口映射（macOS 不支持 `network_mode: host`）。

**Q: OAuth 授权失败？**
确保通过 `http://127.0.0.1:8045` 访问。如果需要远程授权，设置 `ABV_PUBLIC_URL` 环境变量。

**Q: 如何查看实际的 API Key？**
```bash
docker logs antigravity-manager 2>&1 | grep "API Key"
```

**Q: Claude Code 怎么接入？**
```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:8045/v1"
export ANTHROPIC_API_KEY="test"
claude "你好"
```

**Q: 如何彻底删除所有数据？**
```bash
docker rm -f antigravity-manager
rm -rf ~/.antigravity_tools
```
